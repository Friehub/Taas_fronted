# Configuration Reference

The TaaS Gateway is configured via a TOML file — typically named `config.toml` — generated by `hot-core init`. All fields can also be overridden by environment variables using the pattern `SECTION__KEY` (double underscore as separator).

---

## File Location

By default the gateway looks for `config.toml` in the current working directory. Override the path with the `--config` flag:

```bash
hot-core --config /etc/taas/config.toml start
```

---

## `[identity]`

Controls the node's on-chain and cryptographic identity.

| Field | Type | Default | Description |
| :--- | :--- | :--- | :--- |
| `chain_id` | integer | — | Target blockchain chain ID. `1` for Ethereum mainnet, `31337` for Hardhat/Anvil local. |
| `rpc_url` | string | — | JSON-RPC HTTP endpoint for reading on-chain state and submitting transactions. |
| `private_key` | string | — | Ethereum private key as a hex string prefixed with `0x`. Used only if `encrypted_key_path` is not set or the encrypted file does not exist. Prefer the Vault approach — this field is a fallback. |
| `encrypted_key_path` | path | — | Path inside the Vault directory to the AES-GCM encrypted Ethereum key file. Set by `hot-core init`. If this file exists, `private_key` is ignored. |
| `registry_address` | string | — | Ethereum address of the deployed `TruthOracleV3` contract. Used by the Sentinel to subscribe to events and by the Relayer to submit results. |

**Environment override example:**
```bash
IDENTITY__RPC_URL=https://mainnet.infura.io/v3/<key> hot-core start
```

---

## `[server]`

Controls all network listeners and the Admin API.

| Field | Type | Default | Description |
| :--- | :--- | :--- | :--- |
| `rest_port` | integer | `3000` | Port for the public REST API. Handles fetch requests, health checks, and discovery. |
| `grpc_port` | integer | `50054` | Internal gRPC port used for Hot-Core to Sidecar communication. Should not be publicly exposed. |
| `admin_port` | integer | `3001` | Port for the Admin API (metrics, admin commands). Binds to `127.0.0.1` by default. Never expose this publicly. |
| `admin_token` | string | — | Bearer token required for all Admin API requests. Must not be a placeholder. The gateway refuses to start if this is empty or matches a known insecure default. Generated by `hot-core init`. |
| `key_dir` | path | `resources/keys` | Directory where the Vault stores encrypted key files. |
| `wasm_dir` | path | `resources/wasm` | Reserved for future WASM module storage. |

**Environment override example:**
```bash
SERVER__ADMIN_TOKEN=<your-secure-token> hot-core start
```

---

## `[p2p]`

Controls libp2p P2P networking for Mesh Network mode.

| Field | Type | Default | Description |
| :--- | :--- | :--- | :--- |
| `enabled` | boolean | `false` | Enable P2P networking. `false` = Sovereign Node (single operator). `true` = Mesh Network (libp2p GossipSub + Kademlia DHT). |

When `enabled = false`, the binary is built without P2P dependencies (feature-gated). Switching from `false` to `true` requires a rebuild with the `p2p` feature.

---

## `[workers]`

Controls the Sidecar process management and capability configuration.

| Field | Type | Default | Description |
| :--- | :--- | :--- | :--- |
| `spawn_internal` | boolean | `true` | If `true`, Hot-Core starts and supervises the Sidecar process automatically. Set to `false` if you are managing the Sidecar process externally (e.g., in a container sidecar pattern). |
| `internal_worker_count` | integer | `1` | Number of Sidecar instances the Worker Supervisor manages. In most deployments this should be `1`. |
| `logic_host_cwd` | path | `ts/sidecar` | Working directory for the Sidecar process. |
| `logic_host_command` | string | `pnpm run dev` | Command used to start the Sidecar. In production, prefer `pnpm run start` or the compiled `node dist/index.js`. |
| `capabilities_path` | path | `resources/data/capabilities` | Directory containing UCM capability JSON files. `hot-core plugin install` writes to this directory. |
| `enable_tss` | boolean | `false` | Enable BLS12-381 Threshold Signature Scheme. Requires `p2p.enabled = true`. When `false`, signatures use the local Ethereum key only. A `WARN` is emitted at startup if this is `false` in a production-intended configuration. |
| `endpoints` | array | `[]` | Static list of peer endpoints for initial P2P bootstrap. Used in addition to Kademlia DHT peer discovery. |

---

## `[tds]`

Controls the Truth Data Store — Reed-Solomon sharded distributed storage for attestation outcomes and capability blueprints.

| Field | Type | Default | Description |
| :--- | :--- | :--- | :--- |
| `nodes` | string array | `[]` | HTTP endpoints of TDS peer nodes participating in the shard ring. |
| `data_shards` | integer | `1` | Number of data shards (Reed-Solomon `k` value). |
| `parity_shards` | integer | `1` | Number of parity shards. The store can survive `parity_shards` node failures. |

---

## Environment Variables

All config fields can be overridden with environment variables. The format is `SECTION__FIELD` (section and field in uppercase, separated by double underscores). Additionally, the following standalone variables are recognized:

| Variable | Description |
| :--- | :--- |
| `GATEWAY_MASTER_PASSWORD` | Vault decryption password. Preferred over interactive prompt in automated environments. |
| `GATEWAY_PRIVATE_KEY` | Ethereum private key. Overrides `identity.private_key` in the config file. |
| `LOGIC_HOST_TOKEN` | Shared secret for Hot-Core to Sidecar authentication on the internal gRPC channel. |
| `RUST_LOG` | Log level filter for the Rust core. Format: `info`, `debug`, `hot_core=debug`, etc. |
| `LOG_PRETTY` | Set to `true` for human-readable log output (development). Default is structured JSON. |
| `PLUGIN_MANIFEST_PATH` | Override the path to `plugin-manifest.json` used by the Sidecar. |
| `TICKERS_PATH` | Override the path to the authority ticker manifest (`tickers.json`) used by the Sidecar. |

## Security Boot Checks

The gateway performs three mandatory security checks during startup. Any failure is a hard error — the process exits immediately:

| Check | Trigger | Resolution |
| :--- | :--- | :--- |
| Admin token is a placeholder | `admin_token` is empty, equals the known insecure default, or starts with `CHANGE_ME_`. | Run `hot-core init` to generate a secure token, or set `SERVER__ADMIN_TOKEN`. |
| Private key is the Hardhat test key | The normalized key matches `ac0974bec39a17e36ba4a6b4d238ff944bacb478cbed5efcae784d7bf4f2ff80`. | Run `hot-core init` to generate a real node identity. |
| TSS disabled in workers config | `workers.enable_tss = false` — this is a `WARN`, not a hard error. | Set `enable_tss = true` for production Mesh deployments. |

## Minimal Example

```toml
[identity]
chain_id = 1
rpc_url = "https://eth.llamarpc.com"
registry_address = "0xYourTruthOracleV3Address"

[server]
rest_port = 3000
admin_port = 3001
admin_token = "your-generated-uuid-token"

[p2p]
enabled = false

[workers]
spawn_internal = true
logic_host_cwd = "ts/sidecar"
logic_host_command = "node dist/index.js"
capabilities_path = "resources/data/capabilities"
enable_tss = false
```
